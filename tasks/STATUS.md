Current reference-study rule:

- `gamedesign/knowledge/reference_deconstruction.md`
- `AGENTS.md`
- `AI_PIPELINE.md`
- `.codex/skills/primary-gdd-pipeline/references/reference-research-playbook.md`
- Central refs now require a four-pass study before implementation: source
  packet, player transcript, systems extraction, and translation gate with the
  exact native screenshot/scenario proof.
- Central gameplay refs also require an auditable deconstruction doc in the
  active design folder with a source matrix: source quality, checked date, what
  each source proves, what remains uncertain, gameplay footage or a long
  screenshot sequence for interaction claims, and current native capture
  mismatch. If the doc cannot answer "did you study the ref?", implementation
  is blocked until the study is improved.
- Central gameplay refs now start with an observation ledger: at least 5
  timestamped/framed beats covering visible screen state, player action,
  visible response, reward/UI feedback, and inferred meaning. Conclusions and
  implementation come after the ledger, not before it.
- "Make it like X" or "this does not match the ref" feedback now triggers the
  Agent Action Sequence in `gamedesign/knowledge/reference_deconstruction.md`:
  state the reference question, collect sources, record evidence, transcribe
  visible actions, extract systems, translate safely, compare against the
  current native build, and name the next proof screenshot/scenario.
- Reference study now has explicit modes: quick check, central deconstruction,
  and deep deconstruction. Quick checks are only for narrow UI/visual/wording
  questions and cannot unlock core gameplay, economy, balance, primary controls,
  or broad art-direction work. Named gameplay/art drivers default to central
  deconstruction; release-critical pacing, one-hour balance, child-test UX, or
  retention/monetization pressure require deep deconstruction.
- A gameplay reference may only be called "studied" when the durable
  deconstruction records the source packet: links/paths, checked dates,
  gameplay video/walkthrough or long screenshot evidence, official/store/trailer
  visuals when available, supporting guide/review/deconstruction sources for
  balance/friction claims, timestamped/framed observations, current native
  capture, and mismatch audit.
- Reference study now starts with a Reference Lock before implementation or
  final art: study mode, reference question, durable doc path, source packet,
  current native capture plan/path, no-coding/no-final-art boundary, and
  expected native proof. The lock opens only after the durable doc records
  observed source facts, current-build mismatch, borrow/avoid/copy-risk, and
  one scoped next pass tied to screenshot/scenario evidence.
- Reference Intake now comes before opinions or another pass: after a named ref
  or "not like the ref" challenge, the agent must state the exact reference
  question, study mode, durable doc path, required source packet, current native
  capture plan/path, no-coding/no-final-art boundary, and first proof
  screenshot/scenario. Later claims must be labeled observed, inferred,
  user-provided, or unknown; unknown/inferred claims cannot drive
  implementation or final art without a recorded narrow exception.
- Reference study now has a Definition of Ready and explicit stop behavior:
  if mode, durable doc path, Reference Lock, source matrix, current native
  capture, observation ledger, borrow/avoid/copy-risk, current-build mismatch,
  or next native proof are missing, agents must state the study is not ready
  for implementation and keep gathering/deconstructing sources instead of
  coding or generating final art from the reference.
- Reference study must now produce a user-inspectable Reference Digest before
  implementation or final art resumes: study mode, sources checked, 3-5
  observed facts, current-build mismatch, borrow/avoid/copy-risk, and the next
  native screenshot/scenario proof. If the digest cannot be written from the
  durable doc, the reference is still not ready.
- Central/deep reference study now requires a Source Ladder before conclusions:
  user-provided material, official/store/trailer visuals, raw gameplay video,
  walkthrough, or long screenshot evidence, then guides/reviews/lectures/
  deconstructions/wikis/community notes as supporting interpretation. The
  durable doc must record source role, checked date, watched/read scope, what
  each source proves, and whether it is observation evidence or secondary
  interpretation.
- Central/deep reference study now requires a Reference Evidence Board before
  conclusions: cited frames/screenshots for first screen, first input, visible
  response, reward feedback, upgrade/progression UI, and friction/blocked
  state, plus raw gameplay/walkthrough timing evidence. Guides, reviews,
  lectures, wikis, community notes, and deconstructions are supporting
  interpretation for balance/pacing/friction claims, not replacements for
  observed gameplay frames.
- Parallel reference work is research-only until unlocked: agents may gather
  sources, frames, visible-action transcripts, and current native mismatch
  captures beside unrelated setup, but reference-driven gameplay/UI/economy/
  balance/final-art implementation waits for the digest, mismatch audit,
  borrow/avoid/copy-risk, and next native proof.
- `gamedesign/meme-evolution/cow_evolution_deconstruction_v2.md` has been
  audited against that Definition of Ready. It is ready for field-first screen
  grammar, reward placement, and copy-risk translation, but not ready for exact
  first-minute timing, one-hour balance, retention, or monetization pacing
  without a separate deep video/player transcript.
