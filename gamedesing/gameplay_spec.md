# Gameplay Spec

Статус: v0.2 для 30-минутного P0.

## One Sentence

Игрок делает `67-жест`, запускает короткие `Дела`, покупает улучшения и видит,
как герой растет с `1/67` до `15/67`.

## First Screen Through Player Eyes

```text
Герой стоит в Дворовой базе.
Над героем: Сила 1/67.
Большая кнопка: Сделать 67.
Next goal: Собери 5 мем-коинов.
Рядом: Банан реагирует маленькой анимацией.
```

## Canonical Actions

Не использовать в P0 player-facing UI: `Поднять силу`, `Работа`, `Смена`,
`Начать смену`, отдельная вкладка `Мемы`.

Использовать:

- `Сделать 67` - главное нажатие;
- `Дела` - таймерные задания;
- `Обучение` - короткая прокачка навыков;
- `Улучшения` - покупки;
- `Дом` - визуальный рост базы.

## Core Loop

```text
Сделать 67
  -> мем-коины
  -> next-goal card
  -> улучшение / дело / обучение / дом
  -> видимый результат
  -> X/67, доход/сек, навык или реакция
  -> новый район/событие/цель
```

## What Affects What

| Система | Что дает | Что игрок видит |
| --- | --- | --- |
| `Сделать 67` | мем-коины, шанс крутости | герой делает жест, Банан реагирует |
| `Мем-коины` | покупки | счетчик и доступные карточки |
| `Сила X/67` | главный прогресс | число над героем, новые районы |
| `Доход/сек` | idle-прогресс | монеты капают сами |
| `Крутость` | реакции/мемные эффекты | новые реакции и стикеры |
| `Навыки` | доступ к делам | locked card меняется на доступную |
| `Комфорт` | дом/offline | база выглядит лучше |

Правило: любой невидимый стат обязан иметь `visibleResult` в
`data/balance.json`.

## City And Deals

`Город` - карта и открытия. Игрок видит 4 района, статус unlock и одну причину
блокировки.

`Дела` - список таймеров. Одновременно в P0 активно только одно дело. Пока дело
идет, большая кнопка `Сделать 67` остается доступной, а таймер висит сверху.

Deep-link:

```text
Район в Городе -> кнопка К делу -> вкладка Дела с нужной карточкой
```

## Next Goal Card

На главном экране всегда одна цель:

- `Собери 5 мем-коинов`;
- `Купи Пакет для мем-коинов`;
- `Купи Кепку 67`;
- `Открой Киоск`;
- `Запусти одно Дело`;
- `Дойди до 15/67`.

Не показывать 2-3 конкурирующие цели на первом экране.

## Micro-Reactions

Крупных событий всего 4, поэтому между ними нужны маленькие реакции каждые
30-60 секунд активной игры:

- Банан дернулся;
- Клубника подмигнула;
- `До 67 ближе`;
- жест стал быстрее;
- новый район мигнул;
- `Дело сделано`;
- герой получил видимый предмет.

Источник для реализации: `data/balance.json -> microReactions`.

## Visible Rewards

Каждая покупка, дело, обучение и выбор события должны менять экран:

- предмет на герое;
- фон/дом;
- companion;
- реакция Банана/Клубники;
- вспышка `X/67`;
- новая кнопка или район.

Источник для реализации: `data/balance.json -> visibleResult`.

## P0 Screen Map

```text
Intro
  -> Main
      -> Город
          -> район
          -> К делу
      -> Дела
          -> таймер
          -> забрать награду
      -> Улучшения
          -> купить
          -> что изменилось
      -> Дом
          -> база / транспорт / команда
      -> Event modal
      -> Mini-final 15/67
```

## Must-Have Fake Shots

- первый экран `1/67`;
- first click `Сделать 67`;
- first purchase;
- status-up `2/67`;
- city map;
- deals timer;
- event with 2 choices;
- home growth;
- mini-final `15/67`.
