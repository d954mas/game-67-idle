# Fantasy Pocket RPG Concept

## Definition Of Done

- Must exist: concept, reference synthesis, first playable slice, economy/UI contracts, visual web GDD, generated fake-shot art, implementation handoff.
- Out of scope now: playable game build, final title, full lore bible, full runtime-ready art pack, monetization design.
- Accepted proof: the first 5 minutes are understandable from docs and site; camp role is clear; refs are translated into borrow/avoid rules; data JSON validates; visual site runs locally.

## Working Title

`Fantasy Pocket RPG` is a placeholder. Rename after the visual hook and protagonist fantasy become sharper.

## One-Sentence Hook

Compact fantasy RPG: the player explores a dangerous province in short expeditions, survives ruins/events/fights, then returns to a safe camp to heal, craft, talk to companions, level up, and prepare for the next route.

## Player Fantasy

You are not managing a town. You are a growing adventurer: go into the wild, find strange relics, survive a fight, come back to the fire changed, make one smart preparation choice, then push deeper into the province.

## Product Target

- Primary: mobile portrait + PC.
- PC is the development and automation harness.
- Mobile expectation: large touch targets, portrait-first readable screens, 2-5 minute useful sessions.
- PC expectation: same loop with wider layout, mouse-first controls, keyboard shortcuts later.

## Design Pillars

### 1. Big Adventure In Short Sessions

The player gets a useful fantasy beat in 2-5 minutes: travel, event choice, fight, loot, camp prep, unlock.

Violates this: long empty traversal, unclear next step, grinding before first reward.

### 2. Exploration First, Camp Supports It

Camp is a safe rhythm between expeditions: rest, craft, talk, level, prepare. Camp must push the player back into the world.

Violates this: rooms, workers, construction timers, settlement economy, camp becoming the main game.

### 3. Premium Fantasy Clarity

The game should look like a simplified beautiful RPG, not a spreadsheet. Every resource and stat needs an obvious reason on screen.

Violates this: survival clutter, hidden math, too many currencies, unreadable loot comparisons.

## Core Verbs

- Choose a map destination.
- Enter a first-person/illustrated location.
- Search, talk, fight, sneak, or use a skill.
- Gain loot/resources/status changes.
- Return to camp from safe states.
- Rest, craft, equip, level up, talk.
- Unlock the next route, quest clue, or dragon mystery beat.

## Agreed Direction

- Close to `The Quest + Pocket Survivor + Skyrim fantasy`, not a full open-world clone.
- Camp is inspired by BG3/NWN2: safe preparation and companion scenes, not base management.
- `Legendary Dragons`-type fantasy is used as a long-term dragon power/mystery reference, not as gacha or pet collection.
- First implementation should prove one short expedition loop, not the whole RPG.

## Open Decisions

- Final title and setting name.
- Combat presentation: current default is simple first-person/illustrated turn-based panel.
- Character creation depth for first slice.
- Exact art style after user reviews the first visual GDD.
