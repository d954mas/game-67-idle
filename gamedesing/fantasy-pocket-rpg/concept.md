# Fantasy Pocket RPG Concept

## Definition Of Done

- Must exist: first concept, reference pack, first playable slice, minimal economy/UI contracts.
- Out of scope now: generated art, visual GDD website, full lore bible, implementation plan.
- Accepted proof for this pass: a concrete GDD package that makes the first 5 minutes, camp role, exploration loop, currencies/stats, and UI surfaces clear.

## Working Title

`Fantasy Pocket RPG` is a placeholder. Rename when the core fantasy is sharper.

## One-Sentence Hook

A compact fantasy RPG where the player explores a dangerous province, survives quests and ruins in short sessions, then regroups at a safe camp to heal, craft, talk to companions, and prepare for the next expedition.

## Player Fantasy

The player is a wandering adventurer growing from local sellsword into a known hero of the province. The fantasy is not "manage a town"; it is "go into the wild, return changed, prepare better, go deeper."

## Platform

- Primary product target: mobile + PC.
- Mobile expectation: readable portrait-first screens, short sessions, low friction.
- PC expectation: same core loop, wider layout, keyboard/mouse support later.

## Design Pillars

### 1. Big Adventure In Short Sessions

The player should finish a useful action in 2-5 minutes: travel to a location, resolve an event, win a fight, loot, or return to camp.

Violates pillar: long empty traversal, unclear next objective, mandatory grinding before the first reward.

### 2. Exploration First, Camp Supports It

Camp is a pause between expeditions: rest, craft, level up, talk, prepare. It must not become a separate base-management game.

Violates pillar: rooms, workers, long construction timers, economy loops that keep the player in camp instead of pushing them back into the world.

### 3. Premium Fantasy Clarity

The game should feel like a simplified beautiful RPG, not a spreadsheet. UI is readable, systems are understandable, and each stat/change has a visible reason.

Violates pillar: dense survival sim clutter, hidden math, too many currencies, unclear equipment comparisons.

## Core Verbs

- Explore map locations.
- Enter events/ruins/encounters.
- Fight or choose an approach.
- Loot items and resources.
- Rest in safe camp.
- Craft or upgrade gear.
- Talk to companions.
- Level up and choose perks.
- Unlock new regions, quests, and dragon-related mysteries.

## Agreed Direction

- The game is closer to `The Quest + Pocket Survivor + Skyrim fantasy`, not a full open-world clone.
- Add a camp inspired by BG3/NWN2: accessible in safe spots, used for preparation and companion scenes.
- Avoid a full guild/settlement builder for the first version.
- Dragon progression is a long-term symbol of hero growth, not a gacha collection layer.

## Open Decisions

- Combat camera: first-person grid turn-based is the current default, but not final.
- Visual style: current default is painted/stylized fantasy with modern readable UI.
- Camp access rule: likely safe map nodes, cleared locations, campfires, inns, and world-map travel.
