# Core Hook Validation

## Goal

Define what makes the game immediately compelling and how to test it before producing lots of content.

## Core Hook

The hook is:

```text
I see a dangerous place, I choose to risk it, I return changed, and the map opens deeper.
```

This is stronger than:

- generic fantasy lore;
- passive resource collection;
- camp management;
- large open world promise without moment-to-moment decisions.

## First 5-Second Read

The first screen must show:

- where I can go;
- what I risk;
- what I want;
- what is locked next;
- that this is fantasy adventure, not city/base management.

## First 60-Second Payoff

The first minute must produce:

- a location entered;
- a choice made;
- a visible reward or stat change;
- a mystery signal: Dragon Omen or relic;
- a next action.

## First 5-Minute Payoff

The first 5 minutes must produce:

- one combat/check result;
- loot;
- camp unlock;
- recovery/prep choice;
- next map node.

## Why The Core Can Scale

The loop scales because each new content unit can change one of these:

- destination fantasy;
- encounter choice;
- enemy/challenge;
- reward;
- camp reaction;
- unlock.

The player does not need a new system every time. They need a new reason to risk the next route.

## Hook Risks

### Risk: Too Generic Fantasy

Signal:

- player says it looks like any RPG.

Fix:

- stronger region motifs;
- clearer dragon-mark iconography;
- more specific companion reactions.

### Risk: Camp Feels Like The Game

Signal:

- player stays in camp and does not care about next route.

Fix:

- reduce camp actions;
- move rewards/unlocks back to exploration;
- make camp output point to map.

### Risk: Survival Feels Punishing

Signal:

- player avoids routes because supplies/health feel like taxes.

Fix:

- increase early recovery;
- make supplies a route choice, not every-action cost;
- show rewards before costs.

### Risk: Dragon Omen Is Just A Number

Signal:

- player does not ask what the relic means.

Fix:

- add visual relic reaction;
- add companion line;
- add map gate that responds to omen.

## Playtest Script

Ask after 5 minutes:

1. What are you trying to do next?
2. What changed after the ruins?
3. Why did camp matter?
4. What do you think Dragon Omen means?
5. Which button or number was unclear?

## Pass Criteria

- Player can name the next route.
- Player understands camp as preparation.
- Player notices Dragon Omen as mystery.
- Player sees at least one clear reward.
- Player wants to click one more destination.
