# Competitor Strategy — web-dressup (Poki locks)

Full research: `sources/2026-07-09-competitor-hit-research.md`.

## Locked product frame

| Lock | Decision |
|------|----------|
| Distribution | **Poki** |
| Art | **2D** only |
| Layout | **Portrait + landscape** |
| Multiplayer | **No** |
| Spectacle | **Fake shows / fake rivals / fake scores** yes |

## North star

Best **single-player** Poki fashion lab: instant 2D try-on + theme + fake runway
juice — session length without netcode or live lobbies.

## How this changes the competitive answer

| Competitor | Their weapon | Our answer (locked path) |
|------------|--------------|---------------------------|
| Dress to Impress | Live vote + real rivals | **Fake Show**: NPC rivals, local stars, runway spectacle |
| Vortella’s | Freeplay → real MP → competition | **Freeplay → Fake Show** only (skip real MP ladder) |
| Portal fillers | Thin catalog + makeover story | Dual layout polish, theme tags, show juice, recolor later |
| Picrew | Export identity | Screenshot of look / podium (no maker platform) |

## Hit formula (Poki, no multiplayer)

1. 0–3s to first equip; **no text modals**.
2. Deep 2D freeplay (layers, later recolor/poses).
3. **Theme → Fake Show → restyle** as second loop.
4. Dual orientation quality (Poki phone + desktop traffic).
5. Weekly content cadence when live.
6. Optional screenshot; never require social login.

## Explicit non-goals

- Real multiplayer / net lobbies / live human voting
- 3D pipeline
- Gacha energy walls on freeplay
- Long spa/story onboarding
- Feature parity with Nikki / Everskies

## Success metrics (Poki)

- Time to first equip
- Avg session length
- Fake Shows started per session
- Restyles after a show
- Portrait + landscape QA pass
- Poki fit-test / Web Fit criteria when submitted
