## Observations

- `observed` The current capture has a workable gameplay overlay: region
  outline, markers, roads, labels, frame, and desktop/phone layouts.
- `observed` The current capture still looks generated by code because the base
  is mostly flat colored polygons, smooth primitive roads, repeated dot stamps,
  hard outlines, and sparse landmarks.
- `observed` In the Legend reference, the map reads without markers: terrain,
  roads, buildings, water, walls, bridges, and fog already explain the place.
- `observed` Professional fantasy-map tools expose separate authoring concepts:
  landmass/terrain, coast/edge treatment, rivers, roads, tree/mountain symbols,
  labels, themes, textures, stamps, and high-resolution exports.
- `observed` Cartography research and tool pages point to geography/terrain
  logic as part of believability: erosion, tectonic/geologic effects, terrain
  generation, and spatial economy.
- `inferred` For this project, final map art should not be generated by the
  runtime drawing API. The runtime should render a painted atlas and calibrated
  interaction overlays.
- `inferred` AI image generation can be useful for the painted base, but it
  should not be trusted for exact gameplay coordinates, labels, route
  topology, or marker placement.

## Current-Build Mismatch

The current map is structurally better than the first graph, but its visual
language is still mostly an annotated diagram:

- land/region shapes are readable as vector masks, not terrain;
- roads look like overlay strokes, not worn paths embedded in the ground;
- landmarks are tiny simplified icons, not miniature places;
- texture is mostly noise/dots, not material transitions;
- there is little value/lighting hierarchy, so the map lacks depth;
- markers compete with the base because the base has too little authored
  detail;
- the surrounding world does not yet create a strong sense of explored vs
  unknown territory.

## Art Pipeline Takeaways

The next pass should use a two-layer production model:

1. **Painted map base.** A high-resolution atlas image with no gameplay markers
   and no labels that must be exact. It contains terrain materials, region
   silhouette, roads, palisade, ruined gate, old mill, water/debris/fog, dead
   trees, shadows, and atmospheric color grading.
2. **Runtime calibration layer.** Code/data owns region polygons, hit masks,
   current/selected highlights, hero position, destination flag, NPC/enemy/
   service/quest markers, tooltips, and labels.

This is the key shift: code stops drawing the final art. Code aligns gameplay
semantics to an authored image.

## Required Visual Layers

Before accepting another screenshot, the atlas base should contain these layers
or their merged equivalents:

1. `terrain_materials`: ash dirt, dead grass, mud road, stone/wood settlement,
   ruined mill ground, darker unknown edge.
2. `large_value_shapes`: vignette, fog banks, shadowed slopes/ditches, warm
   settlement light, cold outer wasteland.
3. `roads_and_crossings`: worn road with broken edges, wheel ruts, bridge/gate
   crossing, blocked road debris.
4. `landmarks`: Last Post palisade, gate, watch fire, old mill silhouette,
   small ruins, warning signs, dead trees.
5. `region_readability`: an organic region silhouette that follows geography,
   not a generic polygon.
6. `unknown_world_context`: dim neighboring terrain under fog so the map feels
   larger than the playable slice.
7. `overlay_spacing`: clear zones where hero/current marker, POI markers, and
   labels can sit without covering the landmarks.

## Safe AI-Art Use

If generation is used, prompt for a **marker-free painted map base** only:

- top-down fantasy RPG region map;
- dark ash borderland, palisaded last outpost, road to broken gate, old mill,
  dead trees, fog, muted earth/ash palette, painterly game-map texture;
- no UI, no labels, no icons, no text, no characters, no exact Legend
  ornament.

Then postprocess locally:

- crop and color-grade into the atlas aspect ratio;
- add deterministic region masks and hit polygons;
- overlay exact POI markers from `locations.json` / map metadata;
- verify desktop and phone screenshots.

Do not ask the image model to place named NPCs, enemy icons, text, or clickable
regions. That data must stay in code/content.

## Borrow / Avoid / Copy-Risk

- Borrow: map-as-place, geography-first readability, visible roads, landmarks,
  material texture, fogged unknown surroundings, painted base plus precise
  overlay.
- Avoid: continuing to thicken vector lines, adding more marker icons, or
  polishing the frame while the base remains flat.
- Copy-risk: exact Legend map layout, exact icon silhouettes/colors, ornate
  frame, hero/flag/timer art, text labels, and any proprietary fantasy-map tool
  assets without project-specific license/provenance.
