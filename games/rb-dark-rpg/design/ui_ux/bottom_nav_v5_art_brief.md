# Bottom Nav V5 Art Brief

Date: 2026-07-04

Status: candidate for artlead review, not production-sliced.

## Why V5

V4 fixed the wide-frame issue, but became too uniform. The better direction is:
same runtime size, different visual identity per button.

## Keep

- Equal button dimensions and hitboxes.
- Clear icon well and label strip.
- Runtime text only; no baked labels.
- Dark Roblox-like fantasy browser RPG material language.
- 9-slice-friendly frame bases.

## Change From V4

- Each button gets its own material personality.
- Icons use different colors, not one all-gold set.
- Role reads should be obvious by silhouette and color:
  equipment, journal, map, place, more.

## V5 Acceptance

- Buttons are the same size but do not feel like the same skin repeated.
- Icons remain readable when small.
- Icon colors differ enough to help quick recognition.
- Frames do not eat the icon field like v3.
- The set still feels like one game UI family.
