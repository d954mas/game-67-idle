# Синтез четырёх ревью: item-система + state (2026-07-06)

Четыре независимых Opus-агента: (1) адверсариал item-дизайна v2,
(2) индустрия item-систем, (3) жёсткое код-ревью features/game-state,
(4) индустрия сейв-систем. Вердикты: item-дизайн — ship with fixes
(хребет держит, инкремент 1 пока не отдать агенту); game-state —
NEEDS-SURGERY (ядро генератора хорошее, половина фичи мертва).

## 1. Триангулированные находки (нашли ≥2 агента независимо)

**T1. Числовой тип валют — решение №1, нашли все четыре.**
int64 в дизайне: (а) не эмитится текущим генератором (int32, count в
шаблоне капнут 999999); (б) через cJSON-double теряет точность выше 2^53;
(в) экспоненциальная идл-экономика пробивает 9.2e18 в первую сессию
(Cookie Clicker = double ~1.8e308, Antimatter = bignum); (г) целочисленный
add обнуляет дробное производство (0.5/сек × кадр → 0). РЕКОМЕНДАЦИЯ:
валюты = double (решает в,г и JSON-точность), дискретные стаки = int64
(в JSON — строкой). Тип в формате сейва = one-way door, решить ДО кода.

**T2. Атомарность записи инвертирована (нашли #3 и #4).**
Канонический game_storage_save_json пишет fopen("wb") поверх живого файла
(краш = смерть слота), а ПРАВИЛЬНЫЙ рецепт (temp + MoveFileEx
REPLACE|WRITE_THROUGH) уже написан в генерированном game_state_save — но
подключён к дебажному unsafe_path. Фикс: перенести replace_file в
game_storage.c (атомарность = обязанность L0 по практике всех движков)
+ .bak-слот (~15 строк; фолбэк при не-парсе примари).

**T3. Web-сборка не сохраняет ВООБЩЕ (#4; #3 подтвердил обходной путь).**
Emscripten = MEMFS (RAM), сейв умирает по refresh. Для веб-идл студии —
блокер продукта. У rb-dark-rpg в ЕГО копии бэкенда уже есть localStorage-
ветка — консолидировать в шелл-бэкенд (два бэкенда за одной сигнатурой;
на вебе граница = syncfs/IndexedDB-транзакция, не rename).

**T4. Половина game-state мертва (#3, эмпирически проверено).**
generate_state.py содержит ДВЕ реализации; Python берёт последние def'ы —
весь путь маркеров/.c.in (933 строки) + ~700 затенённых строк = ~1600
мёртвых строк. Миграционная машинерия живёт ТОЛЬКО в мёртвой половине
(живой код требует version == текущей, чейна нет — F2). Миграционный файл
rb-dark — сирота с чужим контентом, не компилится, спасён тем, что CMake
его не включает (F3). REQUIRED_FIELDS тоже мёртв (дизайн-док §9 ссылался
на него — устарело).

**T5. Прогрессия: небезграничные циклы за кадр (#1).**
Случайный каскад A→B→A и auto-трек с эмиссией ≥ своей цены вешают кадр.
Фикс: жёсткий кап глубины каскада и auto-итераций на трек/кадр,
drop-with-warn.

**T6. Wall-clock saved_at отсутствует (#2 и #4).**
Оффлайн-прогресс и рег жизней по таймеру требуют персистентного
timestamp. В конверт сейва (шелл); Δt = max(0, now-saved_at) с капом.

## 2. Резолюция несущего противоречия (персистентность)

Адверсариал (#1): §6 «хуки руками» vs §9 «существующий codegen» —
несовместимо, и оба не соответствуют реальному генератору. Код-ревью (#3)
даёт ответ: **(a-lite) пер-фрагментная эволюция**:
- Монолит «game-state как фича» РАСТВОРЯЕТСЯ: бэкенд + конверт +
  оркестрация + JSON-хелперы + транзакционный swap + codegen-инструмент =
  L0 «shell persistence toolkit»; фрагмент + типизированный <id>_state_t +
  save/load-хуки + локальные миграции = у каждой фичи.
- Мелкие фрагменты (settings) пишутся РУКАМИ над общими хелперами;
  генератор получает режим --fragment <id> для больших (items,
  progression: uniformity под агентский парк + схема нужна редактору).
- Codegen ОСТАВИТЬ (вердикт #3 и #4: семейство protobuf, оправдан
  синергией с тулингом) — но одним generic-путём, после ампутации.

## 4. Дешёвые правила сейчас (#1, #2)

- Bootstrap контейнеров: стартовый список (рюкзак, purse) в items.json;
  вместимости объявлены; accept-policy = enum {any, currency_only,
  slot_filter, capacity_1}.
- Порядок load: version-steps → reconcile → карантин; карантин =
  персистентный limbo-список, переживает round-trip, возвращается при
  реставрации def.
- Op-слой validate: запрет rename отгруженного def_id (rename =
  delete+create = сироты в сейвах) + неймспейс-конвенция id; линт «код не
  ключуется по display_name» (страховка урока арт-strcmp).
- Автосейв-топология (предрешить, строить при первом идле):
  debounced-on-dirty + visibilitychange/pagehide на вебе (не
  beforeunload) + saved_at в конверте; hot-path = PrintUnformatted.

## 8. РЕШЕНИЯ ЛИДА (2026-07-06)

Р1. РЕШЕНО: int64 сейчас везде; для экспоненциальных идлов позже — флаг/
    отдельный тип/фича (точка расширения, не строим). Сноски реализации:
    большие счётчики в JSON — строкой (cJSON double, порча выше 2^53);
    дробное производство — аккумулятор в game glue (паттерн в скилл).
Р2. РЕШЕНО: каждая фича отвечает за свои миграции (пер-фрагментные
    версии+чейны); мёртвая монолитная машинерия удаляется. Уточнения:
    стейт игры = тоже фрагмент (features.game); новая фича в апдейте =
    НЕ миграция (нет ключа → дефолты); кросс-фрагментный переезд =
    редкий шаг уровня игры по сырому документу до пофрагментных load,
    версия игры в конверте.
Р3. РЕШЕНО: web-персистентность через localStorage за сигнатурой
    game_storage; ветку rb-dark консолидировать в шелл.
Р4. РЕШЕНО: saved_at (wall-clock момента записи, НЕ игровое время) в
    конверте сейва, владеет шелл; Δt с клампом отрицательных и капом.
Р5. ОЖИДАЕТ: settings руками над хелперами; --fragment к
    items/progression. (рекомендация = да)
Р6. РЕШЕНО: ОБА примитива — set_level (кейс лида: сильный герой в
    прологе, потом силу забирают) и reset для престижей (сброс threshold
    обнуляет внутренний xp).
