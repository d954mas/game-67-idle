# Event system: описание и план (лид зафиксировал 2026-07-06; финальное ревью pending)

Единая система событий фич: типизированное в игре, самоописанное для
тулинга. Вечерняя итерация лида поверх item_system_design §10 (журнал/
события) — ЭТОТ док главнее §10 в транспортной части.
Контекст: templates/design/feature_architecture_2026-07-06.md;
features/game-state/references/state_system_design_2026-07-06.md.

## 1. Решения лида (2026-07-06, вечер)

- События = факты фич; глаголы (команды с reason) отдельно — см.
  item_system §10 (там без изменений).
- **type_hash вместо god-enum**: тип события = nt_hash64 имени
  ("items.txn", "progression.levelup"); объявляется в хедере/схеме СВОЕЙ
  фичи; центрального реестра нет; debug-assert коллизий на старте.
- **Кадровый лог во фрейм-арене (вариант D)**: события живут ОДИН кадр;
  payload произвольного размера в арене; история — приватные копии тех
  потребителей, кому она нужна (DevAPI-лента, аналитика).
- **Универсальные типы полей — на уровне СХЕМЫ, не рантайма**: событие
  объявляется в схеме фичи → генератор эмитит типизированный структ +
  emit-хелпер + type_hash + ДЕСКРИПТОР {имя, тип, offset} → DevAPI/лог/
  аналитика рендерят ЛЮБОЕ событие генерически, фичи рендер-кода не
  пишут. («В игре типизированное, DevAPI по типу понимает и показывает».)

## 2. Конверт и транспорт (D)

```c
typedef struct {
    uint64_t   seq;        /* ГЛОБАЛЬНЫЙ монотонный; uint64 (uint32 wraps ~20ч
                              на стресс-объёмах; влезает в паддинг бесплатно) */
    uint32_t   tick;       /* КАДРОВЫЙ счётчик шелла (не wall-clock) */
    nt_hash64_t type;      /* hash имени события; в debug регистрируется в
                              nt_hash_register_label64 → DevAPI видит имена */
    const void *payload;   /* в СОБСТВЕННОЙ арене событий, произвольный размер */
    uint32_t   size;
} game_event_t;
```

Арена — СВОЯ у event-модуля (НЕ общий nt_mem_scratch движка): отравление
на reset реализуемо game-side без правки движка (engine boundary!),
бюджет отвязан от UI, растяжимая (growable backstop — идиом уже в
дереве); границей петель служит кап поколений, не размер арены; 64КБ =
мягкий бюджет-аларм.

- Один shell-owned append-only массив конвертов на кадр + арена payload'ов
  (домашний идиом — UI уже живёт на кадровой арене). Сброс в конце кадра.
- Производители: только фичи/glue через генерённые emit-хелперы
  (`items_emit_txn(...)` — кладёт payload в арену, конверт в лог).
- Потребители дренят В ТОМ ЖЕ кадре, идя по индексу; события, рождённые
  реакциями (каскады), дописываются в конец и достижимы в том же проходе;
  кап поколений за кадр — страховка от петель.
- Переполнение = громкий dev-warn при исчерпании арены (кап размером
  арены, напр. 64КБ), как у UI-арены. Межкадровых колец, курсоров,
  бэклога, «пропущено N» — НЕТ (умерли за ненадобностью).
- reason-строки и составные payload'ы собираются прямо в арену.
**ВЕРДИКТ ресёрчера (2026-07-06): D побеждает, D > B > A > C.**
Прецеденты за: Unreal Lyra GameplayMessageSubsystem (один бас, строковая
идентичность каналов), MS GDK event-based achievements, Factorio-лог,
event sourcing (один лог + проекции). Провал A: забытый поток в merge =
тихое исчезновение событий фичи из аналитики. Бонус D: реакция в том же
кадре (Bevy платит кадром задержки за двухкадровый буфер).

**Три ОБЯЗАТЕЛЬНЫХ стража D:**
1. **Двухфазный кадр**: фаза ЭМИССИИ (апдейты фич + реакторы-потребители
   доходят до фикспойнта: перепроход по индексу, пока не родится ничего
   нового, под капом поколений) → затем фаза ЗАПИСИ (чистые рекордеры —
   аналитика/лог/DevAPI — один линейный проход). Наивный одиночный
   интерливинг НЕВЕРЕН: рекордер, прошедший раньше реактора, навсегда
   теряет каскадные события кадра (второго кадра у нас нет).
2. **Payload позиционно-независим — str-поля это ОФФСЕТЫ, не указатели**
   (фикс дыры финального ревью: memcpy-retain оставлял бы висячие
   строки). str-поле = uint32 байт-оффсет от начала payload; строки
   копируются emit-хелпером ИНЛАЙН после структа в той же аллокации;
   доступ через генерённый аксессор `(char*)p + p->reason`. Правило:
   арену событий пишут ТОЛЬКО emit-хелперы; каждый str-аргумент
   копируется безусловно (никаких предположений «это статический
   литерал»). Тогда event_retain = корректный слепой memcpy
   (payload не содержит внешних указателей). + отравление арены на
   reset в debug (висячий доступ падает громко).
3. **Выравнивание**: event_emit бампает арену по _Alignof payload'а.

Идл-оффлайн: догон эмитит АГРЕГАТЫ (одно offline_gold +N, не N
транзакций); backstop — растяжимая арена (в дереве уже есть идиом).

## 3. Схема событий и генератор

Событие объявляется в схеме фичи (рядом со стейт-фрагментом):

```json
"events": {
  "levelup": { "fields": {
    "track": { "type": "str" },
    "old":   { "type": "int" },
    "new":   { "type": "int" } } }
}
```

Типы полей (универсальный набор, ИМЕНА ВЫРОВНЕНЫ со стейт-схемой —
`float`/`string`, не f32/str, чтобы агент не расщепил словари): `bool,
int, i64, float (РЕШЕНО: f64 по умолчанию — идл-магнитуды, аналитика
считает в double), string (инлайн-оффсет), hash (event-only; рендер по
label-таблице, enum-как-hash благословлён — rarity=hash("Epic")),
bytes (РЕШЕНО лидом: ОСТАЁТСЯ — «сейчас ничего не стоит»; страж:
рекордеры/аналитика игнорируют, DevAPI показывает размер+hex, скилл
называет цену)`. Линт грамматики имён в генераторе:
`feature.noun_verbed` прошедшее время (дисциплина трекинг-планов);
идиома «эмить N событий, не массив» вместо repeated-полей. Правило двух
таксономий: имя события = ЧТО за факт, reason = ПОЧЕМУ экземпляр;
причина никогда в имени (никаких items.shop_buy), тип факта никогда в
reason-словаре.

Генератор эмитит: типизированный структ + emit-хелпер + константу
type_hash + **дескриптор** {field name, type, offset} (protobuf-паттерн:
схема → структ для кода + дескриптор для тулинга). Игровой код читает
`ev->new_level` с проверками компилятора; новое событие автоматически
видно всем generic-потребителям.

Один встроенный generic-тип `log {str}` — для ad-hoc отладочных сообщений
game glue без заведения схемы.

## 4. Потребители

- **Игра (типизированно)**: в своём апдейте идёт по логу кадра, сравнивает
  type с хешами интересных ей событий, кастит через декод фичи, реагирует
  (попап, сундук, квест-триггер). «Подписка/отписка» тривиальны: читаешь —
  подписан, не читаешь — нет; межкадровых курсоров не существует.
- **Игровой лог**: generic-рендер по дескрипторам, каждый кадр, в свой
  буфер/файл. Порядок родной (индекс массива), сортировать нечего.
- **Локальная аналитика**: каждый кадр копирует/агрегирует нужное в свой
  формат (native — файл в build/analytics/, web — память). Внешний
  провайдер = будущий адаптер её же данных.
- **DevAPI**: `game.events.tail` — лента для ботов. Форма по финальному
  ревью: **рендер при копировании** — рекордер devapi-glue в свою фазу
  (арена жива) рендерит события по дескрипторам в JSON-строки и кладёт в
  фикс-ринг (N=256, ≤512Б со срезом-маркером, oldest-evicted). Бот
  спрашивает по сети когда хочет и получает готовые строки — ни доступа
  к дескрипторам, ни вопросов времени жизни на момент запроса. Сырые
  variable-size payload'ы НЕ рингуются. Экономический срез
  (game.economy.*) = типизированный фильтр поверх той же ленты.
- Экономический журнал items из item_system §10 СТАНОВИТСЯ типом события
  в этом логе (`items.txn {def_id, delta, reason}`) — отдельного кольца в
  items больше нет; git-правка §10 при doc-sync.

## 5. Правила и анти-баги

- События транзиентны: не сейв, не источник истины (истина = стейт;
  потребитель, всё пропустивший, ресинкается снапшотом стейта).
- **ГЛАВНЫЙ агентский анти-баг: не хранить указатели в арену дольше
  кадра** (payload умирает на reset). Правило в скилл + debug-канарейка:
  арена в debug заливается паттерном при reset — use-after-frame виден
  сразу.
- Эмитить события может фича О СЕБЕ (items — про транзакции, progression —
  про лвлапы); игра — через generic `log` или свои game-события в схеме
  фрагмента game.
- Реакции могут эмитить (каскад в том же кадре, под капом поколений);
  бесконечные петли ловятся капом с dev-warn.

## 6. План инкрементов (входит в T0327/T0328 оркестровку)

E1. Шелл: кадровый лог + арена + emit/walk API + кап поколений +
    debug-канарейка. (~200 строк C, ноль зависимостей от генератора.)
E2. Генератор: секция events в схеме фрагмента → структ + emit +
    type_hash + дескриптор. (расширение --fragment из state-плана A4.)
E3. DevAPI: game.events.tail с приватным рингом-копиром + generic-рендер
    по дескрипторам.
E4. Локальная аналитика-писатель (формат лида) + встроенный тип log.
Порядок: E1 можно с A3 state-плана (шелл); E2 после A4 (генератор v2);
items/progression рождаются (T0327 п.5) уже с событиями в схемах.

## 7. Решения лида (2026-07-06, вечер-2)

- Q1 ПРИНЯТО: двухфазный кадр. Механика: агрегатор получает два новых
  явных списка game_features_react() / game_features_record() (7 фазовых
  функций вместо 5 — правка feature_architecture §2); шелл гонит react
  до фикспойнта (кап поколений 16, per-frame-total, dev-warn с именами
  событий последнего круга), затем record одним проходом, затем reset.
  Фазовый debug-assert: emit в фазе RECORD падает с именем события.
  Симметрия ошибок: рекордер-в-react безопасен (перезовут, s_pos
  защищает от повтора), реактор-в-record ловится assert'ом — тихой
  потери нет ни в одной комбинации. Весь цикл — код ШАБЛОНА из коробки
  («сразу сделано как нужно в шаблоне» — лид); игра добавляет
  потребителя ОДНОЙ строкой в один из двух списков.
- Q2 ПРИНЯТО: оффлайн-догон = агрегатные события (одно offline_gold +N);
  растяжимая арена = backstop гигантского кадра.
- Q3 РЕШЕНО: float = f64 по умолчанию.
- Q4 РЕШЕНО: bytes остаётся (лид: «сейчас ничего не стоит») со стражем
  (рекордеры игнорируют, DevAPI size+hex, цена названа в скилле).
- Сигнатуры фаз: react/record несут World* как остальные пять фазовых
  функций; события всегда через event_log(&count) (лог глобален шеллу).

## 8. Итоги финальной пары ревью (2026-07-06): SHIP WITH FIXES — внесены

Адверсариал: дыра event_retain (висячие строки) закрыта позиционно-
независимыми payload'ами (§2 страж 2); двухфазность привязана к
агрегатору (Q1); seq=uint64 (uint32 wrap ~20ч на стресс-объёмах, uint64

render-at-copy (§4); словарь типов выровнен со стейт-схемой; bytes
вырезан. Индустрия: дескриптор = минимум FlatBuffers reflection —
верно; nt_hash_register_label64 УЖЕ в движке → генератор регистрирует
имена типов в debug (DevAPI показывает «items.txn», не хекс);
enum-как-hash благословлён; линт имён feature.noun_verbed; правило двух
таксономий (имя = ЧТО, reason = ПОЧЕМУ; причина не в имени, факт не в

центральный реестр — осознанно не нужны.
