## Method Takeaways

- **Deconstructor of Fun pattern (secondary).** A good teardown is
  evidence-based and covers the core loop, meta/economy, retention drivers, and
  monetization, not just a feature list. Study published deconstructions of a
  specific game as worked examples before claiming to have deconstructed one.
- **Emotion / engagement (Sensemitter, observed).** Beyond systems, deconstruct
  why players disengage: onboarding timing, friction, and the moment-to-moment
  emotional arc. This pairs with the project teachability + core-loop gates.
- **Fun = learning/mastery (Koster, secondary).** A loop is fun when it keeps
  teaching the player something to master. A screen that looks like a game but
  teaches/offers nothing is not fun. This underpins `not-fun = not-done`.

## How This Grounds Our Method

`reference_deconstruction.md` already demands a source matrix, an observation
ledger (>=5 beats), screen grammar, economy/numbers, borrow/avoid/copy-risk, a
mismatch audit, and evidence labels. These external sources confirm that bar:

- a deconstruction needs observed gameplay evidence; Deconstructor of Fun works
  from actual play, not store copy;
- it must reach economy + retention + emotional arc, not just mechanics;
- fun is a real, judgeable property (Koster), so the core-loop gate is not
  optional polish.

Honesty note for our studies: when research is from web text, wiki, or
walkthrough with no first-hand video frames, label claims `secondary` or
`inferred` and mark the source packet incomplete. Do not write `observed` for
evidence we did not see.
