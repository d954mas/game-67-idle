# Creative Intake Playbook

Load this file only when the user's taste, meme anchor, visual target, gameplay
fantasy, or acceptance criteria are unclear.

## Goal

Extract enough direction to produce a first useful visual/gameplay proof without
turning the session into an interview. Ask less, infer more, and label
assumptions.

## Intake Loop

1. Read local context and prior decisions first.
2. Restate what is already known in 3-6 bullets.
3. Ask at most 3 questions only for decisions that would change the output.
4. Convert answers into concrete constraints.
5. Write unresolved items as assumptions, not hidden guesses.
6. Move to a fake shot or first-slice proof quickly.

## High-Value Questions

Prefer questions that force tradeoffs:

- Which single reference is closest, and what exactly should be borrowed?
- What must the player understand in the first 5 seconds?
- What would make you say "this is not my game"?
- Should the first proof optimize for gameplay clarity, meme impact, or art finish?
- What is forbidden: too realistic, too mobile-ad, too clean, too childish, too dark?
- Which element must survive every iteration: number, gesture, character, betrayal, city, home, job, UI density?

Do not ask all of these. Pick up to 3.

## From Vague Taste To Constraints

Translate vague feedback into production constraints:

- "more life sim" -> show city/home/work, stats, job action, lifestyle upgrade.
- "more gameplay" -> add primary action, currencies, upgrade, next goal, feedback.
- "more meme" -> put the meme symbol/gesture in UI, animation, result, or progression.
- "not art for game" -> separate fake shot from prepared/game-use assets and composed proof.
- "not my style" -> ask for closest/avoid refs and regenerate only after restating delta.

## Review Packet Question

After showing a first visual direction, ask one compact question:

```text
Choose the direction: keep this, adjust [specific elements], or regenerate from [specific alternate ref].
```

Avoid broad "what do you think?" prompts. They create unfocused iterations.

## Stop Rules

Stop and ask for a concrete decision when:

- two visual iterations miss the same taste target;
- the user rejects the core fantasy;
- no closest reference is known and visual style is the main deliverable;
- the meme anchor is unclear but central to the game identity.
