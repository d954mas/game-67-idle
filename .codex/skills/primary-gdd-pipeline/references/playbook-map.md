# Primary GDD Playbook Map

Load this when the entrypoint says to choose a focused deep playbook. Load only
the playbook needed for the current gate.

## Core And Intake

- `gdd-core-gates.md`: Definition of Done, folder ownership, Reference
  Intake/Definition of Ready, durable reference deconstruction, loop budget,
  stage gates, validation, and Report Shape.
- `creative-intake-playbook.md`: taste capture, max 3 questions, assumptions,
  unclear acceptance, and early user review.
- `design-stewardship.md`: editing or reviewing existing design docs, keeping
  docs aligned with current gameplay, and avoiding drift.

## Research And Knowledge

- `reference-research-playbook.md`: comparable games, store/review/UI patterns,
  Source Ladder, Reference Evidence Board, Parallel reference work,
  Reference Digest, current-build mismatch audit, and the "not ready for
  implementation" stop.
- `knowledge-capture-playbook.md`: reusable knowledge capture and source notes
  that belong outside the current project wiki.

## Gameplay, Visuals, And Data

- `gameplay-systems-playbook.md`: player verb, first-slice test, core loop,
  challenge, economy, progression, UI states, and machine-readable
  `data/core_loop.json`, `data/ui_flow.json`, `data/asset_manifest.json`, and
  `data/combat.json`.
- `visual-proof-playbook.md`: fake shot, screen grammar, art bible, runtime
  asset checklist, visual/runtime evidence, and mismatch audit.
- `web-gdd-site-playbook.md`: visual GDD site or editable docs surface when web
  work is explicitly in scope.

## Handoff And Review

- `implementation-handoff-playbook.md`: implementation plan, first playable
  slice handoff, build/test discovery, and acceptance gates.
- `quality-review-playbook.md`: mechanics-depth audit before calling a slice
  game-ready.
- `studio-gdd-patterns.md`: production handoff patterns and studio-style
  organization.
- `output-templates.md`: repeated templates.
- `skill-eval-playbook.md`: eval prompts and rubric after skill changes.
