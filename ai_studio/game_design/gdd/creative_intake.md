# Creative Intake

Load this only when taste, meme anchor, visual target, gameplay fantasy, or
acceptance criteria are unclear.

## Goal

Extract enough direction to produce a useful first visual or gameplay proof
without turning the session into an interview.

## Loop

1. Read local context and prior decisions first.
2. Restate what is already known in 3-6 bullets.
3. Ask at most three questions only for decisions that would change the output.
4. Convert answers into concrete constraints.
5. Write unresolved items as assumptions.
6. Move to a fake shot or first-slice proof quickly.

## High-Value Questions

Pick only the few that matter:

- Which single reference is closest, and what exactly should be borrowed?
- What must the player understand in the first five seconds?
- What would make you say "this is not my game"?
- Should the first proof optimize for gameplay clarity, meme impact, or art
  finish?
- What is forbidden: too realistic, too mobile-ad, too clean, too childish, too
  dark?
- Which element must survive every iteration: number, gesture, character,
  betrayal, city, home, job, UI density?

## Taste Fields

Capture accepted taste decisions durably:

- closest refs;
- borrow;
- avoid;
- meme anchor;
- UI density;
- camera/framing;
- art finish;
- no-go palette;
- gameplay clarity requirement;
- visual acceptance test.

## Stop

Ask for a concrete decision when:

- two visual iterations miss the same taste target;
- the user rejects the core fantasy;
- no closest reference is known and visual style is the main deliverable;
- the meme anchor is unclear but central to identity.
