## 1. Вердикт в трёх абзацах

Это настоящий, связный canvas-редактор класса Figma с дисциплинированными костями, а не slop.
Обратная связь громкая и правильная (ошибки ПЕРСИСТЯТ до закрытия, никогда не глотаются); один жест —
один undo проведён везде через журнал; **tool parity реально соблюдён, а не декларативен** (CLI имеет 3
ссылки на `ops.` и ноль записей состояния — обе клиентские поверхности и тесты идут через одни 64 op-функции);
warm python worker инженерно крепкий; живой недеструктивный cleanup-preview — лучшая интеракция в файле.
Видно, что шрамы лида зажили в привычки: зарезервированная ширина кнопок, `scrollbar-gutter: stable`,
`event.code` вместо key (Ctrl+Z переживает кириллицу), `expectHead`-guard, additive-only схема. Это активы
категории «не трогать».

Реальный долг — не в скорости на сегодняшнем масштабе (плоские 200 картинок = 0.04 мс JS/кадр; вечерний
тормоз — это коробка, не страница). Долг в другом: **(a)** слой хранения всё ещё исходит из ОДНОГО писателя,
хотя chat-панель (T0242) и отдельный процесс CLI сделали его многописательным — это настоящая дыра в
безопасности данных, а не гигиена; **(b)** кластеры дублирования, где «единственное предписанное место»
по факту не единственное (codex/agy-spawn ×5, color-distance ×5, border-key ×5, element-guard ×35);
**(c)** store лежит на **YandexDisk-синхронизируемой** папке, и каждый жест выгружает в облако ~0.5 МБ;
**(d)** промахи по felt-quality против мышечной памяти Figma (нет стрелочного nudge, Export погребён под
AI-секциями, карточки спрятаны за правым кликом по пустоте); **(e)** key_matte — ежедневный движок — тратит
62% времени на работу, которую сам же затирает.

Одной фразой: сильный честный инструмент перерос два собственных допущения (один писатель; превью на
исходных пикселях) — почини write lock сейчас, собери дешёвые выигрыши следующим спринтом, а архитектурную
переработку откладывай, пока N реально не вырастет.

## 2. Что хорошо (консолидировано, «не трогать»)

- **Громкие ошибки от края до края.** Тосты-ошибки персистят и не глотаются `[UX,Vis]`; python-refusal
  доезжает одной чистой строкой до `errors.jsonl` и 400, неожиданные баги всё ещё дают traceback `[Py]`;
  remedy-именование в CLI называет точную команду-починку (undo/redo/promote/python-missing) `[CLI]`.
- **Один жест — один undo + дизайн журнала.** Multi-delete/move/paste/dup/batch-alpha сворачиваются в одну
  журналируемую операцию; тонкий append-журнал + жирные sidecar-снапшоты, redo по наибольшему seq,
  compaction держит скан O(cap) `[UX,API]`.
- **Tool parity реально держится.** Каждая мутирующая op завязана и на CLI-verb, и на api-route, обе —
  тонкие маршаллеры `[API,CLI]`; canvas-tools переиспользуют image-модули дословно, без второй реализации
  матта `[Py]`.
- **Explicit-over-magic отказы с причиной В ПАНЕЛИ.** Rotated/flipped серит Detect/Slice/Alpha и показывает
  ту же фразу, которой отказывает op-слой; Distribute с всегда-видимой подсказкой `[UX]`.
- **Дисциплина стабильности лэйаута.** Reset рендерится-но-disabled, чтобы −90/+90 не уезжали; hover-таргеты
  резервируют ширину; `scrollbar-gutter: stable` на каждой панели `[Vis]`.
- **Превью как принцип.** Живой debounced cleanup-preview, ничего не пишется до Apply; тот же `--preview`-сплит
  в CLI (Cancel бесплатен) `[UX,CLI]`.
- **Кадр уже оптимизирован там, где надо.** rAF-коалесинг drag'ов, кэш drag-rect (убирает reflow),
  structure-signature гейтинг DOM, thumbCache/imageCache `[Perf]`.
- **Python-слой чистый.** Per-tool папки, warm worker (parity структурно гарантирован через `runpy`),
  no-fallbacks закон, alpha-preservation байт-в-байт `[Py]`.
- **Связная база токенов + спокойная тёмная тема; additive-only схема** (старые проекты грузятся без
  миграций) `[Vis,API]`.

## 3. Сквозные темы (то, что не увидел ни один ревьюер целиком)

1. **Эра многописательности пришла с chat-панелью, а слой хранения всё ещё про одного писателя.**
   Page + chat (один API-процесс) + CLI (отдельный процесс) теперь мутируют один `project.json` конкурентно.
   Нет lock/CAS; `nextJournalSeq = lastSeq+1` — read-max-then-increment; `expectHead` защищает ТОЛЬКО
   undo/redo/jump, и страница его не шлёт. Плюс catch-all мапит всё в 400 — страница не отличит «head уехал,
   перезагрузись» (409) от «плохой ввод» (400). Это одна тема: **write lock + surfacing конфликта +
   expectHead со страницы** — три findings из `[API]`, которые надо чинить вместе.

2. **«Превью должно быть дешёвым» — не про cleanup, а про класс операций.** cleanup-preview квантайзит
   полноразмерный 1-4 Мпкс PNG на каждый стоп слайдера `[Perf F1]`, отдаёт base64-в-JSON `[Perf F5]`, а на
   канвасе всегда виден screen-scale. Тот же принцип уже правильно выражен в CLI `--preview`. Превью — это
   view-артефакт: proxy-пиксели + blob-транспорт, полный размер только на Apply.

3. **Кластеры дублирования: «единственное предписанное место» не единственное.** codex/agy-spawn ×5 с
   расходящимся stdin/timeout (500/300/900) `[API F5]`; color-distance-to-key ×5 с ДВУМЯ метриками
   (router — Euclidean, keyer — Chebyshev, расходятся у границы допуска) `[Py F3]`; border-key ×5 (mode vs
   median) `[Py F4]`; element-guard ×35 + temp-dance ×6 `[API F4]`; hex-parse ×6, alpha-split ×2 `[Py F5]`.
   Это не косметика — это латентные расхождения (router routes «opaque», keyer keying иначе).

4. **YandexDisk-синхронизация усиливает КАЖДЫЙ I/O-finding.** Один nudge = ~4 полных парса + 2 записи проекта
   + снапшот ~274 КБ (2× проекта) `[API F2]`; снапшоты растут до ~55 МБ/проект `[API F8]`; whole-project
   ресериализация на каждую мутацию `[Perf F4]`. Каждая запись — это ещё и облачная выгрузка. Локально это
   sub-10 мс, но полоса платится на каждом жесте.

5. **Толстый выходной контракт: актив на API/page, чистый налог на CLI.** `list` дампит все проекты целиком
   (95 КБ overflow), каждая мутация эхом весь проект `[CLI F1,F2]`. На page это оправдано (страница
   ре-рендерит из ответа без GET, `[Perf]` подтверждает applyMutation fast-path). На CLI агенту нужны только
   новые id + новый head. Читающий путь (`list`) чинить агрессивно, пишущий (`--quiet`) — opt-in.

6. **Инспектор: длина, а не порядок.** `[Vis W1]` намерил 3823px / ~4.8 экрана, все секции развёрнуты по
   умолчанию; `[UX W9]` — Export (терминальное действие) погребён под тремя AI-секциями. Одно лечение:
   default-collapse низкочастотных секций.

## 5. Вопросы лиду (только настоящие развилки; каждый с рекомендацией)

1. **key_matte bleed (пункт 7).** Какой инвариант главный — no-hidden-pixels (тогда удаляем bleed+repair,
   2.8× даром, вывод байт-в-байт) или edge-extrude-RGB-под-прозрачным нужен (тогда баг в
   `zero_fully_transparent_rgb`)? **Рек.: удалять** — закон «атлас-пакер сам делает 2px extrude / нет скрытых
   пикселей» уже вытеснил bleed. Сначала golden-тест.
2. **Метрика key-distance (пункт 20).** Унифицировать на Chebyshev по всему репо (нужна рекалибровка
   Euclidean-порогов route) или оставить route Euclidean и задокументировать расхождение? **Рек.:
   унифицировать + рекалибровка** — сейчас router и keyer расходятся у границы допуска.
3. **Снапшоты с синхро-диска (пункт 17).** Перенести `snapshots/`+`journal.jsonl` в локальный не-синхр. кэш
   (режет ~90% облачной churn на жест), пожертвовав портируемостью undo-истории между машинами? **Рек.:
   переносить** — history-портируемость почти не нужна, а churn платится каждым жестом.
4. **Recolor style-accent (внутри пункта 11/14).** Сдвинуть `--style-accent` с cyan на violet, чтобы убрать
   два почти-одинаковых teal'а с разным смыслом (region-cyan vs style-cyan)? Это вкус — юзер переучивает
   «style теперь violet». **Рек.: сдвинуть** — это внутренний инструмент, текущая двусмысленность хуже.
5. **Точка входа AI-карточек (пункт 11).** Один top-bar «+ Card» на намеренно-скудном rail — ок? **Рек.: да**
   — headline-воркфлоу не должен висеть на правом клике по пустоте; меню-путь оставляем.

## 6. Чего НЕ делать (осознанно отклонено)

- **Offscreen static-content cache + delta mutation responses** `[Perf #9/#10, L]` — преждевременно: 200
  картинок = 0.04 мс/кадр. Сам ревьюер помечает как «do-differently, не quick-win». Пересматривать только
  после 400-500 эл, и только после мемоизации дерева (пункт 23).
- **CLI `--batch` режим** `[CLI #10]` — строить ТОЛЬКО если профили покажут серийные image-циклы; сам ревью
  говорит «wire к bench-числам, не строй спекулятивно». subtract-not-add.
- **Флип дефолта CLI на `--quiet`** — полный эхо-проект это тестируемый load-bearing контракт (page
  ре-рендерит из ответа). `--quiet` остаётся opt-in (пункт 24).
- **Табличный инспектор** `[Vis D1-alt]` — ломает single-scroll модель, которой инструмент уже привержен;
