## 1. Вердикт (TL;DR)

1. **Архитектура выбрана правильно — рынок это подтверждает.** Все серьёзные игроки (Roblox Assistant + Cube, Unity AI, Bezi, GDevelop agent) сошлись ровно на твоей схеме: агент с machine-readable контекстом движка + цикл «сделал → сам проверил на рантайме» + типизированные представления вместо свободного кодогена. Schema-first GameState, DevAPI (ui.tree, frame.wait, capture.frame), source-first ассеты с provenance — это то, что Bezi продаёт как продукт, а NetEase называет главным источником своих ~30% эффективности (QA-автоматизация). В Defold-сообществе ничего похожего нет вообще — ты не догоняешь, ты впереди.

2. **Но студия прямо сейчас повторяет диагностированный 24.06 failure mode.** После аудита, который сказал «харнесс итерируется без output-truth якоря», сделано **109 коммитов — все 109 тулинговые, 0 игровых**. `games/` пуст, benchmark-фикстура не построена, сам отчёт аудита погиб в gitignored `tmp/`. За 24 дня: ~86% из 856 коммитов — тулинг; 11 прототипов закрыты за 7 дней (16–24.06) и последние ~6 не оставили после себя ни одной записи в базе знаний.

3. **Качество кода — на удивление высокое, проблема не в нём.** Все тестовые сьюты, которые агенты запускали, зелёные (60+63 asset, 39 runtime, 23 taskboard, 19 harness, 6+10+23 raster2d/python). Лицензии/provenance — настоящая валидация, не бумажки. Роутинг контекста реально дешёвый (~3–4 файла / 200–270 строк до начала работы). Проблема — **пайплайн не замкнут в цикл, который производит игру**.

4. **Разорванные звенья цепи (главные находки):**
   - `pull.mjs` **не копирует файлы** ни для одного не-3D ассета (текстуры, UI, аудио, шрифты) — манифест пишется, файла нет, exit 0. Канонический флоу «библиотека → игра» тихо сломан ровно для тех ассетов, которые производит новый raster2d. *(подтверждено мной лично: pull.mjs:141)*
   - Шаг **Promote в новом asset_prep UI отключён** — пайплайн подготовки ассета заканчивается ZIP-ом в tmp, а не ассетом в игре.
   - **Visual gate удалён** (close_slice, visual_axes, art_contract runner — коммиты 28–30.06); правило art-direction-fit заменено на анти-placeholder. Сегодня «некрасивый, но не заглушечный» экран проходит все проверки. Памятка «visual gate DONE и hardened» устарела относительно master.
   - `iterate.py` **«доказывает» на протухшем бинарнике**: build_if_stale() проверяет только существование exe. Оба прекомпилированных exe в репо сегодня падают на smoke — один тихо, с пустым логом.
   - **Шаблон никогда не рождал игру**: создан 24.06, все 11 прототипов старше него. Его схема состояния — на ~80% мёртвые поля Ember Road (~2500 строк мёртвого сгенерированного C на каждую новую игру), настройки/аудио/сейвы не связаны между собой, game_audio.c (303 строки) даже не компилируется, wasm — полускелет при web-first движке.

5. **Церемония с измеренным весом:** `tree.json` (4382 строки, ручной) — в 64 из 109 коммитов; `validation-report.json` — в 61; 6 браузер-серфейсов с копипастнутой навигацией; kanban на ~1900 строк для стора, в котором 0 задач; тест инварианта карты **красный на master** (никто не запускает — CI нет).

6. **Рынок про то, что шипается:** все успешные AI-игры (CODEX MORTIS 81% на Steam, Catvivors, Void Balls) — процедурные/системные жанры (bullet heaven, roguelite, auto-battler), где баланс проверяется кодом. Idle — в этом классе. Главная причина плохих ревью — **некогерентный AI-арт** (source-first + палитра-констрейнт подтверждены рынком; Steam с июня 2026 требует AI-теги, и disclosure по арту даёт измеримый штраф к ревью, а AI-код освобождён). «Фан» не автоматизировал никто — везде человек.

7. **Следующий шаг — не «ещё неделя харнеса».** Ревью само по себе породило ~12 TOP-рекомендаций тулинга — исполнение их всех подряд воспроизведёт диагностированную болезнь. Нужен план с жёстким бюджетом: короткий список исправлений разорванных звеньев + игра как единственный источник новых требований к харнесу.

## 5. Внешний контекст

### 5.1 Коммерческий ландшафт (mid-2026)
Рынок разделился на 4 яруса: (1) **consumer prompt-to-game** (Rosebud, Astrocade $68M, Upit, Websim) — реальные, но мелкие браузерные игры, выживают на ремиксах/шаблонах/кредитах, экспорта из walled garden нет; (2) **агенты внутри движков** (Roblox Assistant+Cube 3D/4D, Unity AI, Bezi, GDevelop) — единственный credible-ярус: project-aware агент правит типизированное представление и сам тестирует; (3) **генерация ассетов/аудио** — production-grade (Meshy, Scenario style-lock, ElevenLabs SFX/TTS); музыка — юридически самый рискованный класс (Suno в судах); (4) **world models** (Genie 3, Muse/WHAM, обещания xAI) — демо-ярус, минуты когерентности, не трогать архитектуру. Big-studio ground truth: Tencent/NetEase — AI окупается в ассетах, кодогене и особенно QA (~30% эффективности, не 10x), дизайн и фан — люди.
**Твоё структурное преимущество**: экспорт и владение (wasm+native, свои ассеты со строгими лицензиями) — ровно то, чего нет у всех consumer-платформ. Unity убила отдельную подписку Muse → AI — фича студии, не продукт.

### 5.2 Open-source / комьюнити практика
Шипаются: CODEX MORTIS (Steam EA, 81%, 12 апдейтов за 19 дней), Catvivors, Void Balls (8 параллельных Claude Code агентов, 10 дней, ~$50/мес) — **все процедурно-системные жанры**. Команды, которые шипают, используют **5–8 ролевых агентов с человеческими гейтами**; 49-агентный оргчарт (22.5k звёзд) не выпустил ни одной игры. OpenGame-Bench формализовал твою задачу верификации: 3 оси — Build Health / Visual Usability / Intent Alignment, скриншот+VLM; ablation показал, что **жанровые шаблоны-скелеты дают +10.1 Build Health**, а **debug-циклы выходят на плато после ~3 итераций**. CMU (807 репо): выигрыш скорости исчезает к 3-му месяцу, сложность +41% навсегда — нужен человек-владелец архитектуры и регулярные упрощающие проходы. Steam: обязательные AI-теги с июня 2026, штраф ревью сконцентрирован в AI-арте, AI-код освобождён. Defold-сообщество: нечего заимствовать, твой DevAPI впереди.

### 5.3 Best practices агентных харнессов
Консенсус 2025–26 твой харнесс в основном уже реализует (repo-as-environment, короткий индекс, subagent-контракты, hooks > проза, замкнутые проверки). Главные пробелы: (1) **evals-as-CI** — 3–5 фикстурных задач с известным хорошим финалом, прогоняемых при каждом изменении промптов/тулов/скиллов (Anthropic «Demystifying evals»); (2) **разделение генератора и оценщика + sprint contracts** — критерии «done» фиксируются до имплементации, оценивает свежий контекст («агенты уверенно хвалят свою посредственную работу»); (3) **харнесс — 90-дневный артефакт**: каждый компонент компенсирует слабость модели, устаревающую с апдейтами; квартальный deletion review. Контр-сигнал: топ-практики (Steinberger) работают почти без харнесса — «побьёт ли эту фичу простой параллельный чат?» — хороший тест для каждого компонента. LLM-judge — advisory, first-tier — rules-based проверки (компиляция, state assertions, pixel checks): ровно твоя «gates advisory, lead backstop» позиция.

### 5.4 Ландшафт asset-пайплайнов
CC0-библиотеки стали API-инфраструктурой (Poly Haven, ambientCG — публичные JSON API: source-first шаг можно автоматизировать). Пиксель-арт: Retro Diffusion — единственный consent-trained генератор (~$0.015/спрайт), PixelLab — скелетная анимация/повороты. UI-иконки: Recraft V3 (нативный SVG + Style Lock). 3D-пропсы: Tripo (скорость/топология), Meshy (авто-риг), Sloyd/Asset Forge (параметрика — чистейший provenance). Аудио: ElevenLabs SFX/TTS с loop — решённая дешёвая линия; Suno — избегать до конца судов. gpt-image-2 **потерял надёжную прозрачность** → твои локальные key/alpha тулы — постоянная стадия, добавить rembg как ML-fallback. Style-consistency для соло — палитра+референс-дисциплина, не обучение моделей. Provenance становится комплаенсом: EU AI Act Art.50 (машиночитаемая маркировка с 02.08.2026) + Steam-теги → добавить в манифест поля из словаря C2PA (tool, model, ai_generated, tier подписки) и one-command rollup для disclosure-форм.
