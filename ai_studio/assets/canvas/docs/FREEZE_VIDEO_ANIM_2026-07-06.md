## Как разморозить

1. Прочитать этот файл + SETUP-README + design_video_anim_canvas (инк. 2-4
   в конце) + T0265 лог.
2. Выбрать движок:
   a) **Облако (рекомендация ресёрча):** hosted WAN на fal.ai — победитель
      research_video_api (гибрид: черновики локально/дёшево, финалы
      облаком; пилот <$5). Вопрос лиду из VERIFY 00 так и открыт.
   b) **Новые локальные модели** (следить за 12GB-классом; ключевой
      критерий по нашим уликам — НАДЁЖНОСТЬ движения по сидам, не качество
      картинки).
   c) Пересобрать старый стек по SETUP-README (1-2 часа + ~42GB докачки).
3. Точка входа в код: `anim_generate.mjs` — единственный шов канваса к
   видео-стеку; менять только стадию generate (контракт неизменен).
4. Очередь фич после разморозки: стир лида «видео отдельно ДО вырезки»
   (двухфазный Generate video → accept → cut) + inc2 режим-таймлайн +
   различимость типов (T0265 лог 2026-07-06), inc3 FLF (T0266).

## UI при заморозке

Пока флаг стоит, лид не должен видеть мёртвые генерационные поля в интерфейсе:

- **Флаг:** `VIDEO_ANIM_FROZEN = true` в `site/app.js` (комментарий рядом ссылается
  сюда) — переключить на `false`, когда программа разморожена.
- **Скрыто за флагом** (генерация видео-анимации): пункт контекст-меню «Animate
  this image» (image-элемент → новая anim-карточка), пункт «New animation card»
  (пустой канвас), и секция «Animation card» (Motion/Profile/Matte/Seed/Loop +
  Generate) в инспекторе группы — `renderAnim(group, root)` в `site/inspector.js`.
- **Скрыто безусловно** (процедурная анимация — отвергнута лидом 2026-07-05,
  НЕ заморожена): вызовы `renderAnimation(element, root)` для image- и
  text-элементов в `site/inspector.js` закомментированы (код `renderAnimation`/
  `animateElementFromTextAction` не удалён — спит, ждёт другого решения).
- **Осталось без изменений:** `renderFlipbook` (Play/Stop + счётчик кадров —
  офлайн-плеер уже сгенерированных дублей, не зависит от генерации), флипбук-
  бейдж в `workspace.js`, chrome-чип типа на анимационных элементах.
- **Как вернуть:** видео-путь — поставить `VIDEO_ANIM_FROZEN = false` в
  `app.js`; процедурный путь — раскомментировать два вызова `renderAnimation(...)`
  (по одному в image- и text-секциях инспектора) отдельным осознанным решением.
