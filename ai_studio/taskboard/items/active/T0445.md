## What

Make the local Items Workbench fail predictably under bursty or malformed input:
all CLI-backed reads need one bounded admission path with finite execution time,
and icon previews must render into bounded thumbnails instead of allocating a
backing canvas at the source atlas-region size for every row.

## Done when

- [ ] Catalog, item, chart, icon, and preview reads share bounded concurrency,
      bounded queued work, coalescing, and explicit overload/timeout responses.
- [ ] API tests prove distinct request floods cannot create unbounded queued or
      active evaluator work and timed-out children are terminated.
- [ ] Icon metadata rejects invalid crop rectangles and the browser uses a fixed
      thumbnail backing store independent of atlas-region dimensions.
- [ ] Focused API/icon/browser tests, full mandatory verify, and Ubuntu/Windows
      CI pass with QTECH evidence recorded.
