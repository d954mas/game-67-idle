## What

Решение лида 2026-07-07 (важно): шаблон должен уметь собираться под web и
грузить паки БЕЗ ручной настройки, и это должно переезжать в новую игру
из коробки при копировании шаблона.

Контекст: wasm-devapi линк починен (2af7a6cf5, T0328), но живой прогон в
браузере упирается в отсутствие web-упаковки: движок ищет ассет-пак по
вшитому абсолютному file:///-пути, браузер это запрещает — игра не доходит
до установки devapi-shim. Паттерн уже решён для rb-dark
(«web pack streams over HTTP», 5bbbff6f) — перенести в шаблон.

Решение лида: разделение пресетов КАК НА ПК. Человеку — чистые
web-сборки без devapi, релиз И дебаг на выбор (wasm-release +
wasm-debug — зеркало native-release/native-debug); агентам —
wasm-devapi. Все собираются, грузят паки и поднимаются командой
сервера. Внимание: wasm-debug давно не собирался (дерева нет);
санитайзер-фикс 2af7a6cf5 должен покрыть и его линк — проверить.

Скоуп:
1. Web-шелл шаблона: index.html + game.js/game.wasm + пак, отдаваемый по
   HTTP относительным путём (rb-dark паттерн; путь пака в wasm-сборке не
   должен быть абсолютным). Работает для ОБОИХ пресетов.
2. Команда запуска локального сервера — одна команда, которой лид
   корректно запускает web-сборку (правильные MIME для .wasm; смотреть,
   что уже есть у rb-dark/studio_shell — не плодить новый сервер, если
   есть готовый). Умеет оба пресета (человеческий = дефолт для лида,
   агентский — для ботов; разные порты или флаг пресета).
3. Из коробки: games/new_game.mjs копирует шаблон — web-сборка + сервер
   должны работать в новой игре без донастройки и разбирательств.
4. Закрыть хвост T0328: живой shim round-trip — window.__devapi
   submit/poll headless-пробой на wasm-devapi-сборке.

НЕ в скоупе: web-бот как полноценный driver (site+agent parity для
браузерных прогонов) — отдельная карточка, когда дойдём.
