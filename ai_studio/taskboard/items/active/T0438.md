representable invariants. Keep the fresh evaluator and current Snapshot/package
boundaries; do not introduce a daemon, cache, or second schema.

## Done when

      stable structured diagnostics before evaluator setup.

      represent, including stack above uint32 and unique currencies whose cap
      semantics are undefined.
- [x] Standalone package build/header/verify validate the canonical Snapshot
      content hash instead of trusting a caller-supplied digest.
- [ ] Semantic Apply validates and compares a hash set for every manifest,
      module, receipt, and state-schema input under one project writer lock
      before replacing the target source.
- [ ] Receipt sealing uses the same project writer lock, expected input hash,
      unique temporary file, fsync, and atomic replace; concurrent writers
      cannot lose an update.
- [ ] Regression tests prove every reviewed failure before implementation, then
      all Items Python, package, feature-contract, and template build checks pass.
