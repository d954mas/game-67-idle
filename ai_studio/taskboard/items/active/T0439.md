## What

Remove measured redundant Workbench work and obsolete proof coupling after the
correctness fixes are stable. Preserve fresh-process isolation and keep all
responses bounded; no persistent evaluator or speculative cache.

## Done when

- [ ] One canonical focused CLI composition supplies selected-item detail
      (inspect/schema/source/dependencies) from one evaluation, and the server
      coalesces/limits duplicate in-flight catalog requests.
- [ ] Initial catalog/detail/chart loading no longer spawns seven concurrent
      evaluator processes; benchmark evidence records command count, wall time,
      context bytes, and process-tree memory before/after.
- [ ] Icon preview applies explicit file/section/image caps, avoids synchronous
      full-pack reads, and serves the bounded atlas page without embedding base64
      in every catalog JSON response.
- [ ] Production package code no longer imports hashing from the C-array proof
      exporter; one small neutral hash owner is used by both paths.
- [ ] Feature manifest/docs distinguish production outputs from proof fixtures
      and accurately describe Windows full profiling, cross-platform CI, and the
      separate Linux package proof.
- [ ] Workbench, benchmark, package, feature-contract, and Studio verification
      suites pass with no legacy JSON or second-evaluator path.

## Open questions

Select the smallest concurrency cap from the existing measured fresh-worker
RSS; do not create a long-lived evaluator.
