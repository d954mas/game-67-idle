## What

Close the reviewed Workbench write-boundary and asynchronous catalog-scoping
defects without adding a second writer, cache, framework, or token service.
The existing semantic CLI remains the only mutation implementation.

## Done when

- [ ] Studio Shell rejects requests whose Host is not the configured loopback
      host, and Items POST additionally requires exact same Origin plus the
      existing JSON content-type/body contract. Negative API tests cover hostile
      Host, absent/cross Origin, and valid loopback requests.
- [ ] Catalog loads use one monotonic generation so a late A response cannot
      overwrite selected B or trigger B detail with A rows.
- [ ] Preview, Apply, and Undo capture catalog/item/generation before awaiting;
      stale responses cannot mutate current UI state, and inverse patches retain
      their owning catalog.
- [ ] Semantic edit controls expose only valid operation/field/level tuples;
      empty numeric input is rejected rather than converted to zero.
- [ ] Focused unit/API tests pass and a real Workbench smoke has no console or
      network errors.

## Open questions

None. Plan review rejected a per-launch CSRF-token subsystem: strict loopback
Host, exact same Origin, and application/json are the smaller complete boundary.
