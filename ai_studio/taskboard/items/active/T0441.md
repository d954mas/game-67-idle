## What

`templates/template/tools/{game,package_web,portal_evidence}.mjs` (and their
`.test.mjs` files) hardcode a depth-2 studio root: `resolve(gameDir, "..", "..")`
plus the static import
`import { inspectPlatformSdkArtifact } from "../../../features/platform-sdk/scripts/artifact_tools.mjs"`
(`package_web.mjs:20`). For a private game copied to `games/private/<id>`
(depth 3), that import resolves to the non-existent `games/features/...`, so
EVERY `node tools/game.mjs ...` command dies at module load with
`ERR_MODULE_NOT_FOUND` — doctor/test/build/package/verify are all unusable.

The game-side cmake already solved the same problem: `cmake/GameOptions.cmake`
probes for `external/neotolis-engine` at depth 2 then depth 3 to set
`GAME_REPO_ROOT`. The node tools never got the equivalent.

A proven fix exists game-side in `game-not-a-trolley-problem`
(2026-07-17): new `tools/lib/studio_root.mjs` exporting `findStudioRoot(gameDir)`
(same probe as cmake), all `resolve(gameDir, "..", "..")` call sites and
`new URL("../../..", import.meta.url)` defaults switched to it, the static
artifact_tools import replaced with a top-level-await dynamic import from the
probed root, and the `portal_evidence.test.mjs` fixture extended to copy
`lib/studio_root.mjs` + create the `external/neotolis-engine` marker. All 41
node tool tests pass at depth 3, and the depth-2 fixture tests still pass.

## Done when

- [ ] `templates/template/tools/` carries the studio-root probe (port the
      game-side `tools/lib/studio_root.mjs` approach) and no tool or test file
      hardcodes `../../..` / `resolve(gameDir, "..", "..")` for the studio root.
- [ ] From a depth-3 checkout (`games/private/<id>`), `node tools/game.mjs doctor`
      reaches dependency verification instead of crashing at import.
- [ ] Template node tests stay green at depth 2 (templates/template) — the
      fixture layouts in `game.test.mjs` / `portal_evidence.test.mjs` cover the
      probe (engine marker present in fixture roots).
