## Screen Grammar

- camera/framing: mostly fixed 2D browser-RPG screen with an illustrated
  location or town hub as the main read, surrounded by persistent RPG chrome.
- primary play space: the location scene, not a split dashboard.
- click targets: NPC/quest marker, route/destination marker, main quest action,
  and reward/equipment surfaces.
- reward location: bottom log/message belt plus quest panel/reward preview;
  reward should not be buried only in prose.
- primary UI: hero status, quest/NPC focus, route highlight, reward preview,
  and one main command.
- secondary UI: backpack/equipment, bottom icon belt, chat/log, locked
  destination reason.
- density: dense enough to read as browser RPG, but trimmed for Ember Road slice
  1; avoid MMO/VIP/event clutter.

## Visual Composition

- dominant shapes: ornate frame, painted location, parchment panels, icon belts,
  portrait medallions, reward slots, route/destination plaques.
- scale hierarchy: location first, hero/NPC second, quest action third, reward
  and progression always visible.
- mood: heroic fantasy road/town hub, not satire, not strategy-city clutter, not
  flat board-game dashboard.
- art requirement: product pass needs real/generated/artist source art for Old
  Gate, Gate Warden, hero, Road Wolf, ring reward, route/destination markers,
  icons, and reusable UI frame pieces.
- Y-up invariant: all game/UI layout decisions stay Y-up. Renderer, input, and
  DevAPI screen rectangles may convert only at explicit boundaries.

## Borrow / Avoid / Copy-Risk

- borrow: illustrated town hub, persistent ornate RPG frame, right quest/NPC
  rail, destination highlight, objective arrows/markers, bottom reward log,
  hero/equipment identity, enemy/reward preview before action.
- avoid: split admin panels, blank rectangle scene, fake text baked into art,
  detached map panel, excessive daily/VIP/event buttons, city-builder systems,
  social/trade/auction scope, idle/away-time framing.
- copy-risk: exact screenshots, UI ornament shapes, character designs, names,
  faction symbols, icon art, monetization surfaces, and exact layouts from
  Legend, Dragon Eternity, AQW, Shakes and Fidget, or Wartune.

## Mismatch Audit Against Current Build

Current screenshot: `build/captures/iterate_first_screen.png`.

- The screen still reads as programmer art: rectangles, simple bars, flat
  silhouettes, and block buildings.
- Old Gate is not yet a believable illustrated place; it is a colored diagram.
- Gate Warden has no portrait or readable character silhouette.
- The route is more connected than the older detached map, but still lacks
  fantasy destination art, objective highlight, and travel affordance.
- Reward is text-only; there is no ring/item icon, loot pop, or equipment
  surface with visual weight.
- Automated battle is a small text card, not a wolf/encounter preview.
- Browser-RPG density is underbuilt: no meaningful icon belt, inventory/equip
  hint, ornate frame, or bottom reward log style.
- Y-up is currently preserved in code by keeping `UiBox` logical coordinates
  Y-up and converting only at renderer/input/DevAPI boundaries; this must not
  regress.
