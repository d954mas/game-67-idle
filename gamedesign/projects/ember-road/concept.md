# Ember Road Concept

## Lead Input

The lead wants an RPG about a hero: map, quests, items, strong fantasy visual
style and animations, towns and travel, farming, grinding, leveling, and
automated battles. The named taste reference is `Legend: Legacy of the Dragons`.

## Working Pitch

`Ember Road` is a beautiful fantasy hero RPG where the player journeys between
towns and map nodes, follows quests, farms encounters, collects items, and
prepares the hero for automated battles. The player is not clicking attacks each
turn; they make strategic RPG decisions around destination, quest priority,
gear, supplies, and when to push or return.

## First Slice Definition Of Done

- One town screen with hero, quest NPC, stats, quest goal, and next action.
- One map screen or map strip with town and one unlocked forest-road node.
- One quest that sends the hero to fight a Road Wolf.
- One automated battle with readable animation/status feedback.
- One loot/XP/gold result and one item comparison/equip choice.
- One quest claim in town and one visible locked next destination.
- One visual target/fake shot accepted or redirected before broad runtime work.
- One native screenshot/product gate proof before expanding content.

## Assumptions

- Native PC is the first playable target.
- The first slice uses a single hero, not a party.
- Combat is automated after the player starts an encounter.
- The first proof optimizes for readable fantasy RPG UI and the complete loop,
  not a large world.
- Reference is a taste anchor only until the durable reference study is filled.

## Out Of Scope For First Slice

- Multiple towns.
- Party management.
- PvP, guilds, market, mounts, pets, crafting, auction, social systems.
- Long quest chains.
- Full procedural item generation.
- Copying reference IP or using unlicensed visual assets.
